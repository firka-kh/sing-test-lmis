# Журнал Взаимодействия (Workflow) — RU

Документация описывает процесс верификации личности на портале при обращении посетителя в Центр занятости населения (ЦЗН).

## UML Диаграмма Последовательности (ИНН-авторизация)

```mermaid
sequenceDiagram
    autonumber
    participant O as Оператор (Инспектор)
    participant S as Верификационный Портал
    participant P as Посетитель (Смартфон)
    participant B as База Данных / API

    Note over O, P: Инициализация процесса
    O->>S: Вводит ИНН посетителя (с паспорта)
    S->>O: Статус: "Ожидание ввода PIN на устройстве"
    
    S->>P: Отображение формы: Введите PIN-код (6 цифр)
    P->>S: Посетитель вводит PIN (Демо: 123456)
    
    S->>P: Отображение формы: Введите Дату рождения
    P->>S: Посетитель вводит дату (Демо: 15.03.1988)
    
    S->>B: Запрос на сверку: ИНН + PIN + Дата рождения
    B-->>S: Результат (Данные корректны / Ошибка)

    alt СЦЕНАРИЙ: "ДА" (Успех)
        S->>O: Зеленый индикатор: "Верифицирован"
        S->>O: Автоматическое открытие карточки посетителя (ФИО, спец-ть, пособие)
        O->>S: Запрос подписи на документ (Приказ/Заявление)
        S->>P: "Подпишите документ на экране"
        P->>S: Нажатие "Подписать"
        S->>O: Статус документа: "Подписано"
    else СЦЕНАРИЙ: "НЕТ" (Отказ)
        S->>O: Красный статус: "Доступ запрещен"
        S->>O: Карточка остается заблокированной (данные скрыты)
        S->>O: Сообщение: "Данные не совпали"
    end
```

## Ключевые состояния
- **ИНН (TIN)**: Первичный идентификатор для поиска в БД.
- **PIN-код**: Временный ключ доступа (6 знаков).
- **Карточка**: Объект с полными персональными данными (только после верификации).
